§04Output engineering · few-shot · cadena de pensamientoOutput engineering · few-shot · chain-of-thoughtOutput engineering · few-shot · cadeia de pensamento

  • Output engineering. Especifica la forma de salida (tabla, JSON, secciones). El formato es parte del pedido, no un detalle.Specify the output shape (table, JSON, sections). Format is part of the request, not a detail.Especifique a forma da saída (tabela, JSON, seções). Formato é parte do pedido.
  • Few-shot. 1–3 ejemplos de entrada→salida deseada fijan el estándar mejor que diez adjetivos.1–3 input→desired-output examples set the standard better than ten adjectives.1–3 exemplos entrada→saída fixam o padrão melhor que dez adjetivos.
  • Chain-of-thought. Pide "piensa paso a paso / planifica antes de ejecutar" cuando el problema tiene razonamiento intermedio.Ask "think step by step / plan before executing" when the problem has intermediate reasoning.Peça "pense passo a passo / planeje antes de executar" quando há raciocínio intermediário.

§06Anti-patronesAnti-patternsAntipadrões

  • El prompt-novela: tres párrafos de contexto y ningún pedido claro.The novel-prompt: three paragraphs of context and no clear request.O prompt-romance: três parágrafos de contexto e nenhum pedido claro.
  • El adjetivo mágico: "sé excelente, detallado, profesional" en vez de un criterio verificable.The magic adjective: "be excellent, detailed, professional" instead of a verifiable criterion.O adjetivo mágico: "seja excelente, detalhado, profissional" em vez de critério verificável.
  • Sin no-inventar: no decirle qué hacer si faltan datos → alucina con seguridad.No no-invent rule: not telling it what to do when data is missing → it hallucinates confidently.Sem regra de não-inventar: não dizer o que fazer se faltam dados → alucina com confiança.
  • Multitarea: cinco objetivos en un prompt → ninguno se cumple bien.Multitasking: five goals in one prompt → none done well.Multitarefa: cinco objetivos num prompt → nenhum bem feito.
