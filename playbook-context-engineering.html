Ingeniería de ContextoContext EngineeringEngenharia de Contexto

El prompt perfecto falla con contexto pobre. Esta disciplina gestiona qué sabe el modelo cuando responde: qué entra a la ventana, qué se recupera y qué se recuerda.The perfect prompt fails with poor context. This discipline manages what the model knows when it answers: what enters the window, what is retrieved and what is remembered.O prompt perfeito falha com contexto pobre. Esta disciplina gere o que o modelo sabe ao responder: o que entra na janela, o que é recuperado e o que é lembrado.

§01La ventana de contextoThe context windowA janela de contexto

La ventana es finita: todo lo que el modelo "ve" en un turno (instrucciones, historial, archivos, tu mensaje). No es memoria infinita — es un presupuesto que administras.The window is finite: everything the model "sees" in a turn (instructions, history, files, your message). It is not infinite memory — it is a budget you manage.A janela é finita: tudo que o modelo "vê" num turno (instruções, histórico, arquivos, sua mensagem). Não é memória infinita — é um orçamento que você administra.

Principio:Principle:Princípio: señal alta, ruido bajo. Cada token que ocupas con algo irrelevante desplaza algo útil.high signal, low noise. Every token spent on the irrelevant displaces something useful.sinal alto, ruído baixo. Cada token gasto no irrelevante desloca algo útil.

§02Context rotContext rotContext rot

A medida que la ventana se llena de historial largo, intentos fallidos y datos pegados, la atención del modelo se degrada: contradice instrucciones tempranas, mezcla hilos, pierde el foco. Eso es context rot.As the window fills with long history, failed attempts and pasted data, the model's attention degrades: it contradicts early instructions, mixes threads, loses focus. That is context rot.À medida que a janela enche de histórico longo, tentativas falhas e dados colados, a atenção do modelo degrada: contradiz instruções iniciais, mistura fios, perde o foco. Isso é context rot.

Antídotos:Antidotes:Antídotos: empieza un hilo nuevo cuando el actual se ensucia · resume y reinyecta solo lo esencial · usa archivos en vez de pegar.start a fresh thread when the current one gets dirty · summarize and re-inject only the essentials · use files instead of pasting.comece um fio novo quando o atual sujar · resuma e reinjete só o essencial · use arquivos em vez de colar.

§04Soberanía del contextoContext sovereigntySoberania do contexto

Tú decides qué entra y qué no. La soberanía del contexto es controlar tus fuentes de verdad: archivos propios, auditables, versionados — no depender de lo que el modelo "recuerde" de cachés o sesiones previas.You decide what enters and what does not. Context sovereignty means controlling your sources of truth: your own files, auditable, versioned — not depending on what the model "remembers" from caches or prior sessions.Você decide o que entra e o que não. A soberania do contexto é controlar suas fontes de verdade: arquivos próprios, auditáveis, versionados — não depender do que o modelo "lembra" de caches ou sessões prévias.

Regla de oro:Golden rule:Regra de ouro: ante una afirmación importante, el repo local es la verdad; el recuerdo del chat no.for any important claim, the local repo is truth; the chat's recall is not.para qualquer afirmação importante, o repo local é a verdade; a memória do chat não.

§06Anti-patronesAnti-patternsAntipadrões

  • El volcado: pegar 20 páginas "por si acaso" → context rot inmediato.The dump: pasting 20 pages "just in case" → instant context rot.O despejo: colar 20 páginas "por via das dúvidas" → context rot imediato.
  • Memoria implícita: confiar en que el chat "se acuerde" en vez de un archivo.Implicit memory: trusting the chat to "remember" instead of a file.Memória implícita: confiar que o chat "lembre" em vez de um arquivo.
  • Reglas contradictorias: capas que se pisan sin jerarquía clara.Contradictory rules: layers that clash with no clear hierarchy.Regras contraditórias: camadas que se chocam sem hierarquia clara.
